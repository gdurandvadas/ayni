# Signal Schema v1 (`0.1.0`) — Historical Reference

> **Historical only.** This page records the serialized `0.1.0` contract from
> the pinned source. Ayni makes **no current parsing, conversion, migration, or
> compatibility promise** for v1.

The exact schema string is `0.1.0`. Source evidence is
`core/src/signal.rs` at commit
[`12a7c80daffdcaf68bdca107cf8c8b362a9e310b`](https://github.com/gdurandvadas/ayni/blob/12a7c80daffdcaf68bdca107cf8c8b362a9e310b/core/src/signal.rs):
`AYNI_SIGNAL_SCHEMA_VERSION`, `RunArtifact`, and the row/payload types below.
The source's `Default` artifact writes that string; its derived deserializer
does not compare an input `schema_version` to the constant.

For signal meanings and the stable vocabulary, see the [signal-contract
index](../signals.md).

## Serialized envelope

The v1 `RunArtifact` serializes these top-level fields:

| Field | JSON type | Serialization rule |
| --- | --- | --- |
| `schema_version` | string | Present; the default artifact value is `0.1.0`. |
| `rows` | array of signal rows | Present when serialized. On deserialization, a missing field defaults to `[]`. |

There are no v1 metadata, aggregate, threshold-summary, offender-summary, or
failure-summary top-level fields in the pinned source.

## Common row encoding

Each `rows` item has these fields:

| Field | JSON type | Serialization rule |
| --- | --- | --- |
| `kind` | `test` \| `coverage` \| `size` \| `complexity` \| `deps` \| `mutation` | Present. |
| `language` | `rust` \| `go` \| `node` \| `python` \| `kotlin` | Present. |
| `scope` | object | Present; `workspace_root` (string) is present, while `path`, `package`, and `file` are optional strings omitted when null. |
| `pass` | boolean | Present. |
| `result` | tagged result object | Present; its `kind` tag selects the result type below. |
| `budget` | tagged budget value | Present; its `kind` tag selects one of the six variants, each carrying `serde_json::Value`. |
| `offenders` | tagged offender object | Present as `{ "kind": <kind>, "items": [...] }`. |
| `delta_vs_previous` | object | Omitted when null; when present it is `{ "changes": <JSON value> }`. |

`result` is internally tagged with `kind`; `budget` is internally tagged with
`kind`; and `offenders` is tagged with `kind` and `items`. Thus their tag names
are serialized exactly as `kind`. `warn` and `fail` are the serialized `level`
values.

## Row payloads

The following fields are exact field names from the pinned Rust types. “Optional”
means `Option<T>` and is omitted during serialization when null.

### `test`

| Object | Fields |
| --- | --- |
| `result` | `total_tests` (u64), `passed` (u64), `failed` (u64), `duration_ms` (optional u64), `runner` (string), `failure` (optional command failure) |
| `offenders.items[]` | `file` (optional string), `line` (optional u64), `message` (string), `test_name` (optional string) |

### `coverage`

| Object | Fields |
| --- | --- |
| `result` | `percent` (optional f64), `line_percent` (optional f64), `branch_percent` (optional f64), `engine` (string), `status` (string), `failure` (optional command failure) |
| `offenders.items[]` | `file` (string), `line` (optional u64), `value` (f64), `level` (`warn` \| `fail`) |

### `size`

| Object | Fields |
| --- | --- |
| `result` | `max_lines` (u64), `total_files` (u64), `warn_count` (u64), `fail_count` (u64) |
| `offenders.items[]` | `file` (string), `value` (u64), `warn` (u64), `fail` (u64), `level` (`warn` \| `fail`) |

### `complexity`

| Object | Fields |
| --- | --- |
| `result` | `engine` (string), `method` (string), `measured_functions` (u64), `max_fn_cyclomatic` (f64), `max_fn_cognitive` (optional f64), `warn_count` (u64), `fail_count` (u64), `failure` (optional command failure) |
| `offenders.items[]` | `file` (string), `line` (u64), `function` (string), `cyclomatic` (f64), `cognitive` (optional f64), `level` (`warn` \| `fail`) |

### `deps`

| Object | Fields |
| --- | --- |
| `result` | `crate_count` (u64), `edge_count` (u64), `violation_count` (u64) |
| `offenders.items[]` | `from` (string), `to` (string), `rule` (string), `level` (`warn` \| `fail`) |

### `mutation`

| Object | Fields |
| --- | --- |
| `result` | `engine` (string), `killed` (u64), `survived` (u64), `timeout` (u64), `score` (optional f64), `failure` (optional command failure) |
| `offenders.items[]` | `file` (optional string), `line` (optional u64), `mutation_kind` (string), `message` (string), `level` (`warn` \| `fail`) |

## Command failures

Where a result includes optional `failure`, it is omitted when null and otherwise
has `category` (string), `classification` (string), `command` (string), `cwd`
(string), `exit_code` (optional i32), and `message` (string). `exit_code` is
omitted when null.

## Validation behavior in the pinned source

The pinned `RunArtifact` uses derived Serde serialization and deserialization.
Beyond the Rust field and enum types above (and the `rows` default), it declares
no custom v1 envelope validation, summary validation, or schema-version equality
check. It also declares no `deny_unknown_fields` attribute.
