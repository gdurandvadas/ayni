# Signal Schema v2 (`0.2.0`)

The current serialized schema string is `0.2.0`. Source evidence is the current
[`core/src/signal.rs`](https://github.com/gdurandvadas/ayni/blob/main/core/src/signal.rs):
`AYNI_SIGNAL_SCHEMA_VERSION`, `RunArtifactSerialization`, `RunArtifactWire`,
and the row/payload types. Current artifact production and prior-artifact
selection are in `cli/src/main.rs` (`build_artifact_metadata` and
`load_previous_artifact`).

For signal meanings and the stable vocabulary, see the [signal-contract
index](../signals.md). For the historical envelope, see [v1](v1.md).

## Serialized envelope

`RunArtifact` serializes this flat top-level envelope; its internal `metadata`
field is not nested in JSON.

| Field | JSON type | Serialization rule |
| --- | --- | --- |
| `schema_version` | string | Present; `RunArtifact::new` and `Default` set `0.2.0`. |
| `generated_at` | string | Present. |
| `ayni_version` | string | Present. |
| `invocation` | object | Present; `command` (string), `languages` (array of language values; defaults to `[]` when absent on input), and `scope` (optional scope, omitted when null). |
| `output` | object | Present; `format` (string) and `destination` (string). |
| `config_path` | string | Present. |
| `repository_root` | string | Present. |
| `aggregate` | object | Present derived summary described below. |
| `applied_thresholds` | array | Present derived summary described below. |
| `rows` | array of signal rows | Present. |
| `offender_summaries` | array | Present derived summary described below. |
| `failure_summaries` | array | Omitted when no command failures occur. |

Language values are `rust`, `go`, `node`, `python`, and `kotlin`. A scope object
has required `workspace_root` (string) and optional `path`, `package`, and
`file` strings, each omitted when null.

## Derived summaries

`aggregate` contains `status` (`pass` or `fail`), `total_rows` (u64),
`passing_rows` (u64), `failing_rows` (u64), `warning_offenders` (u64), and
`failing_offenders` (u64). Its status is `pass` exactly when every row passes.
Its offender counts treat every `test` offender as failing; for every other
kind, they count `level: "warn"` and `level: "fail"` respectively.

Each `applied_thresholds` item contains `kind`, `language`, `scope`, and
`budget`, copied from each row in row order. Each `offender_summaries` item is emitted only
when that row has at least one offender and contains `kind`, `language`,
`scope`, `total` (u64), `warning_count` (u64), and `failing_count` (u64).

Each `failure_summaries` item contains `kind`, `language`, `scope`, `category`,
`classification`, `command`, `cwd`, optional `exit_code` (i32, omitted when
null), and `message`. It is derived once for each row result with a command
failure.

## Common row encoding

Each `rows` item has `kind`, `language`, `scope`, `pass`, `result`, `budget`,
`offenders`, and optional `delta_vs_previous`. Their exact encoding is:

| Field | JSON type | Serialization rule |
| --- | --- | --- |
| `kind` | `test` \| `coverage` \| `size` \| `complexity` \| `deps` \| `mutation` | Present. |
| `language` | language value | Present. |
| `scope` | scope object | Present. |
| `pass` | boolean | Present. |
| `result` | internally tagged object | Present; `kind` selects the result type below. |
| `budget` | internally tagged value | Present; `kind` selects one of six variants, each carrying `serde_json::Value`. |
| `offenders` | object | Present as `{ "kind": <kind>, "items": [...] }`. |
| `delta_vs_previous` | object | Omitted when null; when present it is `{ "changes": <JSON value> }`. |

The `result` and `budget` tags serialize as `kind`; `offenders` serializes its
tag as `kind` and its collection as `items`. `level` serializes as `warn` or
`fail`.

## Row payloads

“Optional” means `Option<T>` and is omitted when null.

### `test`

| Object | Fields |
| --- | --- |
| `result` | `total_tests` (u64), `passed` (u64), `failed` (u64), `duration_ms` (optional u64), `runner` (string), `failure` (optional command failure) |
| `offenders.items[]` | `file` (optional string), `line` (optional u64), `message` (string), `test_name` (optional string) |

### `coverage`

| Object | Fields |
| --- | --- |
| `result` | `percent` (optional f64), `line_percent` (optional f64), `branch_percent` (optional f64), `engine` (string), `status` (string), `failure` (optional command failure) |
| `offenders.items[]` | `file` (string), `line` (optional u64), `value` (f64), `level` (`warn` \| `fail`) |

`percent` is the primary headline percentage (0–100). The implementation's
headline selection prefers `percent`, then `line_percent`, then `branch_percent`.

### `size`

| Object | Fields |
| --- | --- |
| `result` | `max_lines` (u64), `total_files` (u64), `warn_count` (u64), `fail_count` (u64), `failure` (optional command failure) |
| `offenders.items[]` | `file` (string), `value` (u64), `warn` (u64), `fail` (u64), `level` (`warn` \| `fail`) |

### `complexity`

| Object | Fields |
| --- | --- |
| `result` | `engine` (string), `method` (string), `measured_functions` (u64), `max_fn_cyclomatic` (f64), `max_fn_cognitive` (optional f64), `warn_count` (u64), `fail_count` (u64), `failure` (optional command failure) |
| `offenders.items[]` | `file` (string), `line` (u64), `function` (string), `cyclomatic` (f64), `cognitive` (optional f64), `level` (`warn` \| `fail`) |

### `deps`

| Object | Fields |
| --- | --- |
| `result` | `crate_count` (u64), `edge_count` (u64), `violation_count` (u64), `failure` (optional command failure) |
| `offenders.items[]` | `from` (string), `to` (string), `rule` (string), `level` (`warn` \| `fail`) |

### `mutation`

| Object | Fields |
| --- | --- |
| `result` | `engine` (string), `killed` (u64), `survived` (u64), `timeout` (u64), `score` (optional f64), `failure` (optional command failure) |
| `offenders.items[]` | `file` (optional string), `line` (optional u64), `mutation_kind` (string), `message` (string), `level` (`warn` \| `fail`) |

## Command failures and validation

Each optional result `failure` is a `CommandFailure`: `category` (string),
`classification` (string), `command` (string), `cwd` (string), optional
`exit_code` (i32, omitted when null), and `message` (string).

On deserialization, `RunArtifactWire` requires every top-level field above
except `failure_summaries`, which defaults to absent. The implementation rebuilds
the artifact from canonical metadata and rows, then rejects input unless the
provided `aggregate`, `applied_thresholds`, `offender_summaries`, and
`failure_summaries` exactly equal the deterministic values derived from its
rows. It does not compare an input `schema_version` to `0.2.0` in this core
deserializer and declares no `deny_unknown_fields` attribute.

For delta computation, the CLI accepts a previous artifact only after successful
deserialization and only when its `schema_version == "0.2.0"`; other versions
are ignored. This is not v1 migration or compatibility support.
